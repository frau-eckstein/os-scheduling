Theorie: CPU-Scheduling

Das Betriebssystem verwaltet mehrere Prozesse gleichzeitig. Da die CPU aber zu einem Zeitpunkt nur einen Prozess ausführen kann, muss eine Scheduling-Strategie entscheiden, welcher Prozess als nächstes die CPU erhält. Diese Entscheidung beeinflusst direkt die Reaktionszeit und Effizienz des gesamten Systems.

Das Vier-Zustands-Modell

Jeder Prozess durchläuft während seiner Lebensdauer mehrere Zustände. Das Betriebssystem verwaltet diese Übergänge. Beim Scheduling werden nur bereite Prozesse berücksichtigt – blockierte werden übersprungen.

bereit
Wartet in der Warteschlange auf CPU-Zuteilung. Ist rechenbereit, bekommt aber noch keine CPU.
aktiv
Hat die CPU zugeteilt bekommen und führt gerade Programmcode aus.
blockiert
Wartet auf ein externes Ereignis (z. B. Festplatten- oder Netzwerkzugriff). Wird beim Scheduling ignoriert.
beendet
Alle Verarbeitungsschritte abgeschlossen. Wartet auf Freigabe der Ressourcen durch das Betriebssystem.
Wichtige Begriffe
Prozess
Ein Prozess ist ein laufendes Programm aus Sicht des Betriebssystems. Er hat eigenen Speicher, Variablen und einen Zustand. Unterschied: Schwergewichtige Prozesse haben eigenen Speicherbereich, leichtgewichtige (Threads) teilen sich einen.
Scheduling
Scheduling (engl. to schedule = planen) bezeichnet die Zuteilung von CPU-Zeit an wartende Prozesse. Es ist eine zentrale Aufgabe moderner Betriebssysteme.
Burst-Zeit
Die Burst-Zeit (auch: Ausführungszeit, CPU-Zeit) ist die Gesamtzeit, die ein Prozess tatsächlich die CPU benötigt, um vollständig abgearbeitet zu werden.
Wartezeit
Zeit vom Eintreffen eines Prozesses bis zu seinem ersten Start auf der CPU. Ziel guter Scheduling-Strategien ist eine möglichst geringe durchschnittliche Wartezeit.
Turnaround Time
Zeit vom Eintreffen bis zum vollständigen Abschluss eines Prozesses (Wartezeit + Burst-Zeit). Auch: Durchlaufzeit.
Kontextwechsel
Wechsel der CPU von einem Prozess zu einem anderen. Der Zustand des alten Prozesses wird gespeichert, der neue wiederhergestellt. Verursacht Overhead (Zeitverlust).
Preemption
Bei preemptive Scheduling kann das Betriebssystem einem laufenden Prozess die CPU jederzeit entziehen. Bei non-preemptive Scheduling läuft ein Prozess bis er fertig ist oder sich selbst blockiert.
Starvation
Verhungern (Starvation) tritt auf, wenn ein Prozess so lange wartet, dass er nie oder erst sehr spät ausgeführt wird.
Quantum (RR)
Feste Zeitscheibengröße beim Round-Robin-Verfahren. Zu klein → viele Kontextwechsel. Zu groß → verhält sich wie FCFS. Typisch: einige Millisekunden.
Multitasking
Das Betriebssystem lässt mehrere Programme scheinbar gleichzeitig laufen, indem die CPU sehr schnell zwischen ihnen wechselt. Scheduling macht dies erst möglich.
Zeiteinheit (ZE)
Abstrakte Einheit für Zeitangaben in Scheduling-Beispielen. In der Praxis entspricht eine ZE oft einer bestimmten Anzahl von CPU-Taktzyklen oder Millisekunden.
