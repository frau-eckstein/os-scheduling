🗃️ 3.1 · Das Verzeichnis /proc

Bei Linux-Betriebssystemen ist /proc eine wichtige Kernel-Schnittstelle. Die Dateien und Unterverzeichnisse in diesem Verzeichnis werden dynamisch vom Kernel generiert, d. h. sie sind nicht im Hintergrundspeicher abgelegt.

Sie enthalten verschiedenste Informationen über den Zustand und die Konfiguration des Betriebssystemkernels und sind z. B. mittels cat direkt auslesbar.

Beispiel: /proc erkunden
$ cat /proc/cpuinfo
processor    : 0
vendor_id    : GenuineIntel
cpu family   : 6
...

📜 3.2 · Kernel-Nachrichten lesen

Mit dmesg können Statusmeldungen des Kernels eingesehen werden. Je nachdem, in welcher Form und in welchem Format die Ausgabe benötigt wird, lässt sich das Ausgabeformat über Kommandozeilenparameter entsprechend anpassen.

dmesg – menschenlesbare Ausgabe
$ dmesg --help    # Hilfe anzeigen
$ sudo dmesg -H -T  # Menschenlesbar + Zeitstempel
[Mo Jan 1 12:00:00 2024] Linux version 5.15 ...
[Mo Jan 1 12:00:01 2024] CPU: Intel Core i5 ...
ARBEITSAUFTRAG 2 Analyse von dmesg
a)

Erkundigen Sie sich durch Eingabe des Befehls dmesg --help über die verfügbaren Kommandozeilenparameter und deren Bedeutung.

b)

Wählen Sie geeignete Parameter aus der Auflistung von a) aus, sodass die Ausgabe von Kernelnachrichten möglichst menschenlesbar erfolgt.

c)

Führen Sie den Befehl sudo dmesg aus und ergänzen Sie dabei die gewünschten Kommandozeilenparameter aus b) am Ende des Befehls (jeweils durch ein Leerzeichen getrennt).

d)

Analysieren Sie die Ausgabe von c) und versuchen Sie die Bedeutung der einzelnen Zeilen bestmöglich selbst zu erschließen (ggf. gemeinsam in Kleingruppen).

e)

Fassen Sie allgemein zusammen, welche Art von Informationen ein Aufruf von dmesg liefert und welchem Zweck diese Ausgabe dienen kann.

f)

Stellen Sie eine Hypothese auf, weshalb die "menschenlesbare" Ausgabe nicht der Standardfall ist und dieses Ausgabeformat somit erst explizit durch Kommandozeilenparameter aktiviert werden muss.

🔌 3.3 · Betriebssystemaufrufe / Syscalls

Der Betriebssystemkern hat alleinigen direkten Zugriff auf zentrale Ressourcen (Speicher, CPU, …). Damit jedoch auch Anwendungsprogramme diese Ressourcen nutzen können, bietet der Kernel über sogenannte Syscalls eine Schnittstelle an, mittels der Anwendungsprogramme den Kernel beauftragen können bestimmte privilegierte Operationen auszuführen.

Der Kernel übernimmt daraufhin die Kontrolle über die Ausführung, arbeitet die privilegierten Operationen ab und überträgt die Kontrolle anschließend zurück an das aufrufende Anwendungsprogramm:

📱
Anwendungsprogramm
Braucht Ressource → ruft Syscall auf
⚡
Syscall
Kernel übernimmt Kontrolle → führt privilegierte Operation aus
🔄
Rückgabe
Kontrolle zurück an Anwendungsprogramm
