⚙️ 1 · Grundlagen der Diensteverwaltung

Startet man auf einem Rechner ein Programm, muss das Betriebssystem die zu diesem Programm gehörigen Prozesse verwalten (vgl. Scheduling). Lässt man sich aber z. B. mit dem Befehl ps -aux eine Liste der laufenden Prozesse auf einem Rechner anzeigen, wird schnell klar, dass neben den direkt von den Nutzern gestarteten Prozessen noch eine ganze Reihe weiterer Prozesse vorhanden sind. Ein Teil dieser Prozesse gehört zum Betriebssystem selbst (bei Prozessen, die in der Liste von ps -aux in eckigen Klammern aufgeführt werden ist dies in der Regel der Fall).

Neben Nutzer- und Betriebssystemprozessen gibt es eine weitere Kategorie: Dienste.
Das sind Programme, die das Betriebssystem automatisch verwaltet – z. B. beim Systemstart, ohne Nutzerinteraktion.

Eine besonders zentrale Rolle spielen Dienste bei Rechnern, welche als Server genutzt werden, aber auch auf normalen Heim- und Arbeitsplatz-PCs sind in der Regel verschiedene Dienste aktiv.

👤
Nutzerprozesse
Vom Nutzer manuell gestartet
🐧
BS-Prozesse
Kernel-intern (in [...] in ps)
🔧
Dienste
Vom BS verwaltet, autostart
ARBEITSAUFTRAG 1 Dienste erkunden und verstehen
a)

Lassen Sie sich mit systemctl list-unit-files --state=enabled eine Liste der aktuell auf dem Rechner aktivierten Dienste ausgeben. Finden Sie anschließend für 2–3 dieser Dienste heraus, welchem Zweck sie dienen.

b)

Lassen Sie sich für einen Dienst XXX Ihrer Wahl mit systemctl status XXX.service weitere Informationen anzeigen. Analysieren Sie die Ausgabe und erklären Sie den Unterschied zwischen den Zustandspaaren enabled/disabled und active/inactive.

🌐 2 · Einen Webserver-Dienst einrichten

Im Folgenden soll beispielhaft der Webserver Apache2 als Dienst installiert und gestartet werden. Durch die Einrichtung als Dienst soll der Webserver so auch dann erreichbar sein, wenn der Rechner neugestartet wurde und/oder aktuell kein Nutzer am Rechner angemeldet ist.

Apache2 – Installation und Prüfung
$ sudo apt install apache2
$ systemctl status apache2
  ● apache2.service - The Apache HTTP Server
     Loaded: loaded (/lib/systemd/system/apache2.service; enabled)
     Active: active (running) since ...
  # Webserver testen: http://127.0.0.1:80 im Browser aufrufen
ARBEITSAUFTRAG 2 Apache2-Webserver konfigurieren
a)

Installieren Sie den Apache-Webserver mit sudo apt install apache2. Überprüfen Sie anschließend mit systemctl status apache2, ob der Dienst ordnungsgemäß eingerichtet, aktiviert und gestartet wurde. Falls nicht, holen Sie dies mit systemctl enable –now apache2 nach.
Überprüfen Sie die Funktion des Webservers danach im Webbrowser: Bei Aufruf der URL http://127.0.0.1:80 sollte eine Demoseite des Webservers angezeigt werden.

b)

Ändern Sie die Konfigurationsdatei /etc/apache2/ports.conf so ab, dass der Webserver künftig unter Port 8080 (statt wie bisher 80) erreichbar ist. Prüfen Sie anschließend im Browser, ob die Änderung greift.
Führen Sie dann systemctl restart apache2 aus und prüfen Sie erneut, unter welchem Port der Webserver nun erreichbar ist. Begründen Sie den Unterschied in Ihren Beobachtungen vor und nach dem Neustart.

c)

Ändern Sie die Konfigurationsdatei aus Teilaufgabe b) erneut ab, sodass der Webserver unter dem Port 8080abc erreichbar sein soll. Starten Sie den Dienst anschließend neu und analysieren Sie mittels systemctl status apache2 und journalctl -xe, wie sich derart ungültige Angaben in der Konfigurationsdatei bemerkbar machen.

d)

Überprüfen Sie auf geeignete Weise, mit welchen Benutzerrechten der Webserver standardmäßig ausgeführt wird, und begründe diese Festlegung im Kontext von Datenschutz und Datensicherheit

🛠️ 3 · Einen eigenen Dienst einrichten

Viele oft verwendete Dienste (wie z. B. der Apache-Webserver) sind als fertige Softwarepakete verfügbar und müssen lediglich installiert und ggf. konfiguriert werden. Es ist jedoch auch möglich gänzlich individuelle Dienste im System einzurichten.

Im Folgenden soll dies an einem einfachen Beispiel nachvollzogen werden, indem ein Dienst eingerichtet wird, welcher in regelmäßigen Abständen den Füllstand des Systemlaufwerks in eine Logdatei schreibt.

ARBEITSAUFTRAG 3 Eigenen Dienst programmieren und einrichten
a)

Verfassen Sie (ggf. unter Zuhilfenahme geeigneter Bibliotheken) ein einfaches Programm, das in einer Endloswiederholung:
 ... den aktuell verfügbaren freien Speicherplatz unter dem Dateisystempfad / ermittelt
 ... an einem geeigneten Ort zusammen mit einem Zeitstempel in eine Logdatei schreibt, wobei die Logdatei dabei stets um eine weitere Zeile erweitert (und NICHT der bisherige Inhalt überschrieben) werden soll
 ... anschließend 20 Sekunden wartet, bevor die nächste Iteration der Endloswiederholung durchlaufen wird

💡
Freier Speicherplatz in Java: new java.io.File("/").getFreeSpace() liefert die freien Bytes. In Python: import shutil; shutil.disk_usage("/"). Für den Zeitstempel: LocalDateTime.now() (Java) bzw. datetime.now() (Python).
b)

Erstellen Sie die Datei /etc/systemd/system/meindienst.service mit dem nachfolgendem Inhalt. Ersetzen Sie dabei ... durch den zum Starten Ihres Programms aus a) notwendigen Terminalbefehl.

[Unit]
Description=Mein selbsterstellter Dienst
 
[Service]
Type=simple
ExecStart=...  <-- Startbefehl Ihres Programms
 
[Install]
WantedBy=multi-user.target
c)

Sorgen Sie mittels systemctl daemon-reload dafür, dass die Diensteverwaltung des Betriebssystems auf die soeben neu erstellte Dienst-Konfigurationsdatei aufmerksam wird und aktivieren/starten Sie den selbst erstellten Dienst „meindienst“ anschließend mittels der bereits bekannten Terminalbefehle.

Dienst aktivieren
$ sudo systemctl daemon-reload            # Neue Konfiguration laden
$ sudo systemctl enable --now meindienst  # Aktivieren + starten
$ systemctl status meindienst             # Status prüfen
d)

Überprüfen Sie, ob der Dienst wie vorgesehen arbeitet und auch nach einem Systemneustart weiterhin aktiv ist.

e)

Beenden Sie den zum Dienst gehörigen Prozess manuell mit ps und kill. Überprüfen Sie danach, wie die Diensteverwaltung des Bestriebssystems auf diesen simulierten "Programmabsturz" reagiert. Begründen Sie, weshalb diese Vorgehensweise des Betriebssystems in der Regel sinnvoll ist.

f)

Erweitern Sie Ihr Programm um regelmäßige Terminalausgaben. Finden Sie anschließend heraus und beschreiben Sie, auf welchem Weg diese Ausgaben während der Ausführung als Dienst abrufbar sind.

💡
Tipp zu f): Schauen Sie sich journalctl -fu meindienst an.
