🚀 1.1 · Das System starten und kennenlernen

Ubuntu begrüßt nach dem Start mit dem GNOME-Desktop, einer grafischen Benutzeroberfläche mit Aktivitätenübersicht, Anwendungsmenü und Systemleiste. Vieles davon ist bereits aus anderen Betriebssystemen bekannt – manches ist anders.

ARBEITSAUFTRAG 1 Linux-Distribution erkunden

Arbeiten Sie sich eigenständig in die grundlegende Bedienung von Ubuntu ein. Finden Sie dabei insbesondere heraus …

… wie man das System neustartet bzw. herunterfährt.

… wie man den aktuell angemeldeten Benutzer abmelden kann und anschließend wieder zum Anmeldebildschirm gelangt.

… wie man installierte Programme findet und startet.

… über wie viel Arbeitsspeicher und wie viele CPU-Kerne das System verfügt.

💡
Terminal öffnen:
Drücken Sie Strg+Alt+T oder suchen Sie nach „Terminal" in der Aktivitätenübersicht.
ARBEITSAUFTRAG 2 Vergleich mit anderen Betriebssystemen

Vergleichen Sie Erscheinungsbild, Funktionsumfang und Bedienung des Linux-Betriebssystems mit einem von Ihnen privat genutzten Betriebssystem (z. B. Windows, macOS, Android, …).

📝

Erstellen Sie darauf aufbauend eine stichpunktartige Gegenüberstellung mit Gemeinsamkeiten und Unterschieden.

📁 1.2 · Dateiverwaltung unter Linux

Eine wichtige Aufgabe jedes Betriebssystems ist die Verwaltung des Hintergrundspeichers. Unter Linux gibt es dafür verschiedene grafische Programme, sogenannte Dateimanager.
Jede Distribution bringt in der Regel einen eigenen mit, z. B.:

📂 Nemo
📂 Dolphin
📂 Thunar
📂 Nautilus
💡
Dateimanager nutzen oder installieren:
Dateimanager können im Terminal einfach mit dem jeweiligen Namen gestartet werden.
Falls ein Dateimanager noch nicht installiert ist, kann er mit sudo apt install ... nachinstalliert werden.
ARBEITSAUFTRAG 3 Dateimanager erkunden und vergleichen
a)

Ermitteln Sie den Standard-Dateimanager von GNOME, indem Sie versuchen, die o. g. Dateimanager über das Terminal zu starten.

b)

Starten Sie nun alle genannten Programme Nemo, Dolphin, Thunar und Nautilus. Falls ein Programm fehlt, installieren Sie es zuerst.

c)

Vergleichen Sie die genannten Programme hinsichtlich Erscheinungsbild und Funktionsumfang.

d)

Diskutieren Sie, inwieweit folgende Aussage zutrifft:

„Die oben genannten Programme sind Teil des Betriebssystemkerns, da sie unter Linux die Aufgabe der Verwaltung des Hintergrundspeichers übernehmen."

🖼️ 1.3 · Desktop Environments

Unter Linux ist die grafische Benutzeroberfläche kein fester Bestandteil des Systems, sondern ein austauschbares Zusatzprogramm, das sogenannte Desktop Environment (DE), wie z. B.:

GNOME KDE Plasma Cinnamon Xfce MATE

Dabei können auch mehrere parallel installiert sein.

⚠️
Hinweis:
Das Installieren mehrerer Desktop Environments kann viele Pakete und mehrere hundert MB Speicherplatz benötigen, daher wird hier darauf verzichtet.
ARBEITSAUFTRAG 4 Desktop Environments vergleichen und einordnen

Recherchieren Sie zu mindestens drei verschiedenen Desktop Environments.

a)

Beschreiben Sie, worin sich die Desktop Environments unterscheiden.

b)

Desktop Environments werden teils für bestimmte Einsatzszenarien entwickelt (z. B. Touchscreen-Bedienung, sehr alte/schwache Hardware, ...).
Erläutern Sie für zwei Desktop Environments Ihrer Wahl, für welchen Einsatzkontext diese vermutlich entworfen wurden.

c)

Diskutieren Sie analog zu Arbeitsauftrag 3c, inwieweit das Desktop Environment Teil des Betriebssystemkerns oder der Anwendungssoftware ist.
